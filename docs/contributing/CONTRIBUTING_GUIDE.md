# Contributing to the Unified Rigidity Framework (URF)

## Purpose
Provide guidelines for researchers or contributors interested in extending,
testing, or analyzing the URF ecosystem.

## Contribution Types

### 1. Theoretical Extensions
Possible directions include:
- new rigidity invariants
- extensions to additional graph classes
- entropy-depth bounds in new models
- spectral or dynamical rigidity analogues

### 2. Formal Verification
Lean formalization targets include:
- structural lemmas
- invariant definitions
- proof dependencies
- canonical theorem statements

### 3. Computational Experiments
Examples:
- random graph experiments
- expansion and cycle-overlap studies
- entropy-depth measurement scripts

### 4. Documentation Improvements
Suggested contributions:
- examples
- diagrams
- exposition improvements
- literature comparisons

## Repository Roles

urf-core
Framework axioms and core definitions.

urf-spine
Canonical structural lemmas and invariants.

chronos-urf-rr
Reference experiments and computational artifacts.

urf-textbook
Pedagogical exposition and worked examples.

## Contribution Workflow

1. Fork repository
2. Create feature branch
3. Add documentation, proofs, or experiments
4. Submit pull request with clear description

## Open Research Directions

- rigidity thresholds on expanders
- locality limits in refinement algorithms
- entropy-depth bounds in alternative models
- connections to spectral rigidity
