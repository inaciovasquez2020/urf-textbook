# Call for Independent Forks and Verification

The Unified Rigidity Framework repositories are open for independent inspection, forking, testing, and adversarial review.

Researchers, engineers, students, and technically interested readers are invited to:

1. fork any repository,
2. run the verification/test suites,
3. inspect the status and frontier documents,
4. open issues for unclear claims or reproducibility failures,
5. submit pull requests that improve clarity, tests, citations, or formalization,
6. build independent derivative experiments or verification layers.

The goal is not endorsement.

The goal is independent reproducibility, stress-testing, and public technical scrutiny.
